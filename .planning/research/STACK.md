# Technology Stack

**Project:** Zestok — Mobile Companion (Android)
**Researched:** 2026-07-07
**Overall confidence:** HIGH (official Capacitor docs verified) / MEDIUM (community posts)

## Recommended Stack

### Core Runtime

| Technology | Version | Purpose | Why |
|------------|---------|---------|-----|
| **Capacitor** | 8.4.1 (stable, 2026-06-19) | Native Android runtime for web apps | Already in deps; is the standard bridge between web code and Android native APIs |
| **Node.js** | 22.x LTS+ | Build tooling, CLI | **Required by Capacitor 8** — will not work on Node 20 or below |
| **npm** | 10.x+ (ships with Node 22) | Package management | Ships with Node |

**Critical:** `@capacitor/android@^8.4.1` is already in `dependencies` in `package.json`. This is correct. You must also install `@capacitor/core` and `@capacitor/cli` — they are **missing** from the current project.

### Android SDK / IDE

| Technology | Version | Purpose | Why |
|------------|---------|---------|-----|
| **Android Studio** | Otter \| 2025.2.1+ | Native Android project management, SDK manager, emulator, build signing | **Required by Capacitor 8** — older versions cause AGP conflicts (see pitfalls) |
| **Android SDK Platform** | API 36 (Android 16) | Compile/target SDK | Latest stable; Capacitor 8 defaults to `compileSdk = targetSdk = 36` |
| **Android SDK Build-Tools** | 36.x.x | APK packaging | Bundled with SDK Platform install |
| **Android Gradle Plugin (AGP)** | 8.13.0 (managed by Capacitor) | Gradle plugin for Android builds | Set automatically by `npx cap add android` |
| **Gradle Wrapper** | 8.14.3 | Build system | Set automatically by Capacitor |

**Do NOT install a separate JDK** — Android Studio Otter bundles JDK 21 automatically, which is what AGP 8.13.0 needs.

### Required npm Packages

| Package | Version | Type | Purpose |
|---------|---------|------|---------|
| `@capacitor/android` | ^8.4.1 | dependency | **Already installed.** Android native runtime |
| `@capacitor/core` | ^8.4.1 | dependency | **NEED TO ADD.** Core JS runtime, includes CapacitorHttp plugin |
| `@capacitor/cli` | ^8.4.1 | devDependency | **NEED TO ADD.** CLI for `npx cap init`, `add`, `sync`, `build` |
| `@capacitor/preferences` | ^8.0.0 | dependency | **NEED TO ADD.** Persistent key/value storage (SharedPreferences on Android) |
| `@capacitor/app` | ^8.0.0 | dependency | **NEED TO ADD.** App lifecycle events, back button handling |
| `@capacitor/splash-screen` | ^8.0.0 | dependency | Optional. Splash screen on app launch |

### Web Build Tooling

| Technology | Version | Purpose | Why |
|------------|---------|---------|-----|
| **Vite** | ^6.x | Web bundler for the mobile app | Zero-config for vanilla JS/HTML; fast HMR; Capacitor community standard |
| **Vanilla JS + HTML + CSS** | — | UI layer | Mobile app is read-only dashboards; no framework complexity needed for v1 |

**Why Vite over a framework (React, Vue, etc.):**
- The mobile companion is a thin dashboard UI (6 screens max: PIN entry, dashboard, stock table, rate check, settings)
- Vanilla JS + Vite keeps the bundle tiny (~50KB vs 200KB+ for framework overhead)
- The developer is familiar with vanilla JS from the Electron app — no framework learning curve
- Capacitor works identically with any frontend; the `webDir` config just points to the build output

## Installation & Setup

### 1. Install Missing Core Packages

```bash
# cd into the mobile app directory (e.g., stock-mobile/)
npm install @capacitor/core@^8.4.1
npm install -D @capacitor/cli@^8.4.1
npm install @capacitor/preferences@^8.0.0
npm install @capacitor/app@^8.0.0
```

### 2. Initialize Capacitor

```bash
npx cap init
```

You will be prompted for:
- **App name:** `Zestok`
- **App Package ID:** `com.usama.stockmanagement` (must match the existing `appId` in package.json's `build.appId`)

This creates `capacitor.config.ts` at the project root.

### 3. Create `capacitor.config.ts`

```typescript
import type { CapacitorConfig } from '@capacitor/cli';

const config: CapacitorConfig = {
  appId: 'com.usama.stockmanagement',
  appName: 'Zestok',
  webDir: 'www',
  server: {
    // Allow cleartext (HTTP) to the backend server during development.
    // The backend runs on HTTP, not HTTPS.
    cleartext: true,
  },
  plugins: {
    CapacitorHttp: {
      // Enable native HTTP library instead of browser fetch.
      // Required for Android to make HTTP requests without CORS issues.
      enabled: true,
    },
  },
};

export default config;
```

### 4. Add the Android Platform

```bash
npx cap add android
```

This creates the `android/` directory with a full Android Studio project. This directory should be committed to version control (it's a platform-specific project, not a build artifact).

### 5. Install Android Studio & SDK

1. Download Android Studio Otter (2025.2.1+) from [developer.android.com/studio](https://developer.android.com/studio)
2. During installation, select:
   - **Android SDK Platform API 36** (Android 16)
   - **Android SDK Build-Tools 36**
   - **Android Emulator** (for testing)
3. After install, open **Tools → SDK Manager** and verify:
   - SDK Platforms tab: `Android API 36` is checked
   - SDK Tools tab: `Android SDK Build-Tools 36` is installed

### 6. Development Workflow

```bash
# Build web assets
npx vite build

# Sync to Android native project
npx cap sync

# Open Android Studio
npx cap open android

# OR run directly on device/emulator from CLI
npx cap run android
```

**The cycle:** Edit web code → `npx vite build` → `npx cap sync` → `npx cap run android`

## Keystore & Signing

### Generate a Keystore (One-Time)

```bash
keytool -genkey -v -keystore my-release-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-key-alias -storetype JKS
```

You will be prompted for:
- **Keystore password:** Generate and store in a password manager
- **Key password:** Can be same as keystore password
- **Distinguished name info:** First/Last name, Org Unit, Org, City, State, Country (2-letter code, e.g., `PK` for Pakistan)

**WARNING:** Losing the keystore or password means you **cannot publish updates** to an existing app on the Play Store. This is irreversible. Back up `my-release-key.jks` to:
- A password manager
- An encrypted cloud backup
- A physical offline drive

**Do NOT commit the keystore to git.** Add `my-release-key.jks` to `.gitignore`.

### Signing Method: CLI (Recommended for CI)

**Recommended configuration for APK (sideloading):**

```bash
npx cap build android \
  --keystorepath ./my-release-key.jks \
  --keystorepass YOUR_STORE_PASSWORD \
  --keystorealias my-key-alias \
  --keystorealiaspass YOUR_KEY_PASSWORD \
  --androidreleasetype APK \
  --signing-type apksigner
```

**For AAB (Google Play Store):**

```bash
npx cap build android \
  --keystorepath ./my-release-key.jks \
  --keystorepass YOUR_STORE_PASSWORD \
  --keystorealias my-key-alias \
  --keystorealiaspass YOUR_KEY_PASSWORD \
  --androidreleasetype AAB \
  --signing-type jarsigner
```

### Signing Method: capacitor.config.ts (Alternative)

For convenience, you can embed signing config in `capacitor.config.ts` using environment variables:

```typescript
import type { CapacitorConfig } from '@capacitor/cli';

const config: CapacitorConfig = {
  appId: 'com.usama.stockmanagement',
  appName: 'Zestok',
  webDir: 'www',
  server: {
    cleartext: true,
  },
  android: {
    buildOptions: {
      keystorePath: process.env.KEYSTORE_PATH,
      keystorePassword: process.env.KEYSTORE_PASSWORD,
      keystoreAlias: process.env.KEYSTORE_ALIAS,
      keystoreAliasPassword: process.env.KEYSTORE_ALIAS_PASSWORD,
      releaseType: 'APK',
      signingType: 'apksigner',
    },
  },
  plugins: {
    CapacitorHttp: {
      enabled: true,
    },
  },
};

export default config;
```

Then run:

```bash
set KEYSTORE_PATH=./my-release-key.jks
set KEYSTORE_PASSWORD=your_store_password
set KEYSTORE_ALIAS=my-key-alias
set KEYSTORE_ALIAS_PASSWORD=your_key_password
npx cap build android
```

### Signing Method: Android Studio (Manual / GUI)

1. `npx cap open android`
2. In Android Studio: **Build → Generate Signed Bundle / APK**
3. Choose **APK** (for sideloading) or **Android App Bundle** (for Play Store)
4. Point to your `my-release-key.jks` file, enter credentials
5. Output is at `android/app/build/outputs/apk/release/app-release.apk` or `android/app/build/outputs/bundle/release/app-release.aab`

## Plugin Usage

### HTTP Requests: CapacitorHttp (Built-in)

The `CapacitorHttp` plugin is bundled inside `@capacitor/core` — no separate install needed. It uses native Android HTTP libraries (OkHttp under the hood) instead of the browser's fetch API.

**Enable in capacitor.config.ts:**
```typescript
plugins: {
  CapacitorHttp: {
    enabled: true,  // Patches fetch/XMLHttpRequest to use native libraries
  },
}
```

**Usage in app code:**
```javascript
import { CapacitorHttp } from '@capacitor/core';

// GET request
const response = await CapacitorHttp.get({
  url: 'http://84.235.249.239:3000/api/entries',
  headers: { 'Authorization': `Bearer ${token}` },
  params: { limit: '50' },
});

// POST request
const response = await CapacitorHttp.post({
  url: 'http://84.235.249.239:3000/api/pin/verify',
  headers: { 'Content-Type': 'application/json' },
  data: { pin: userInputPin },
});
```

**Why use CapacitorHttp instead of browser `fetch`:**
1. No CORS issues — native HTTP does not enforce browser CORS policy
2. More reliable — handles cookies, redirects, and TLS correctly on Android
3. Full timeout control via `connectTimeout` and `readTimeout` options

### Local Storage: @capacitor/preferences

```bash
npm install @capacitor/preferences@^8.0.0
```

**Usage:**
```javascript
import { Preferences } from '@capacitor/preferences';

// Save auth token
await Preferences.set({
  key: 'auth_token',
  value: 'eyJhbGciOiJIUzI1NiIs...'
});

// Read it back
const { value } = await Preferences.get({ key: 'auth_token' });

// Remove on logout
await Preferences.remove({ key: 'auth_token' });
```

**Storage backend on each platform:**
| Platform | Backend | Persistent? |
|----------|---------|-------------|
| Android | SharedPreferences | ✅ Yes |
| iOS | UserDefaults | ✅ Yes |
| Web (PWA) | localStorage | ✅ Yes |

**Why Preferences and not localStorage directly:**
- `localStorage` has no guaranteed persistence on Android WebView — the OS can wipe it on low storage or after app updates
- Preferences maps to native storage APIs (SharedPreferences on Android, UserDefaults on iOS) which are explicitly preserved by the OS
- Preferences has a web fallback to localStorage for dev testing in browser

### App Lifecycle: @capacitor/app

```bash
npm install @capacitor/app@^8.0.0
```

**Usage for session persistence check:**
```javascript
import { App } from '@capacitor/app';

// Listen for app resume (to refresh dashboard)
App.addListener('resume', () => {
  refreshDashboard();
});

// Handle Android back button
App.addListener('backButton', ({ canGoBack }) => {
  if (!canGoBack) {
    App.exitApp();
  } else {
    window.history.back();
  }
});
```

## Alternatives Considered

| Category | Recommended | Alternative | Why Not |
|----------|-------------|-------------|---------|
| **Mobile framework** | Capacitor 8 | Flutter / React Native | Already in deps; reuses JS skills from Electron; read-only app doesn't need native performance |
| **Frontend framework** | Vanilla JS + Vite | React / Vue / Svelte | App is 6 thin screens; framework overhead adds 150KB+ and learning curve not justified |
| **HTTP client** | CapacitorHttp (built-in) | Axios / ky / bare fetch | Native HTTP avoids CORS; no extra dependency; properly handles Android TLS |
| **Local storage** | @capacitor/preferences | localStorage / SQLite | Preferences is the simplest persistent key/value store; SQLite overkill for token/cache only |
| **Signing tool** | apksigner (APK) / jarsigner (AAB) | Android Studio GUI | CLI is scriptable, repeatable, CI-friendly |
| **Package manager** | npm | pnpm / yarn | npm ships with Node; no additional install needed for this developer's first mobile project |

## Pitfalls & Warnings (Details in PITFALLS.md)

1. **Missing `@capacitor/core` and `@capacitor/cli`.** The project only has `@capacitor/android`. You need all three at matching versions.
2. **Do NOT install `@capacitor/ios`.** iOS is out of scope; installing it adds unnecessary complexity for an Android-only v1.
3. **Node.js 22 is mandatory.** Capacitor 8 throws errors on Node 20.
4. **Android Studio Otter 2025.2.1+ is mandatory.** Older Android Studio versions break with AGP 8.13.0 and produce cryptic error messages.
5. **apksigner + AAB = crash.** Issue #8428 — `apksigner` cannot sign AAB files. Use `jarsigner` for AAB, `apksigner` for APK.
6. **jarsigner + APK targeting API 30+ = rejected install.** SHA-1 signatures produced by `jarsigner` are rejected by Android 11+. Always use `apksigner` for APK builds.
7. **`npx cap build android` has a known issue** (Capacitor issue #8106) where the built APK fails to install on devices. The workaround is to use Android Studio or ensure `signingType: 'apksigner'` for APK builds.
8. **Keystore loss is irreversible.** There is no way to recover or replace a keystore for an existing Play Store listing (without Google Play App Signing, which requires advance setup).
