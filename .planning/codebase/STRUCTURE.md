## Directory Purposes

**`src/` — Application Source:**
- Purpose: All application source code for both the Electron app and the Express server
- Contains: Electron main/renderer/preload processes, Express app, HTML/CSS frontend, database layer
- Key files: `main.js` (entry point), `renderer.js` (UI logic), `server.js` (API routes)

**`src/db/` — Database Layer:**
- Purpose: PostgreSQL connection management, schema initialization, and data migration
- Contains: Connection pool factory (`pool.js`), migration script (`migrate.js`), reference schema (`schema.sql`)
- Key files: `pool.js` exports `masterPool`, `getTenantPool()`, `ensureMasterDb()`

**`server/` — Server Bootstrap:**
- Purpose: Production server entry point with environment loading and HTTPS setup
- Contains: Server start script (`index.js`), environment config (`.env`), dependencies (`package.json`)
- Key files: `index.js` — imports `src/server.js` and `src/db/pool.js`, starts HTTP/HTTPS listener

**`build/` — Installer Customization:**
- Purpose: NSIS installer custom NSH macros for Electron Builder
- Contains: `installer.nsh` — cleanup of previous installations, process termination on install/uninstall

**`scripts/` — Utility Scripts:**
- Purpose: One-off maintenance and migration scripts
- Contains: `migrate-pin.js` — moves legacy PIN from `app_settings` table to `tenants` table

**`dist/` — Build Output:**
- Purpose: Generated by `electron-builder`, contains packaged Windows installer and update artifacts
- Contains: `Stock.exe`, `latest.yml`, `win-unpacked/` directory, build config files
- Generated: Yes (by `npm run build`)
- Committed: Yes (release artifacts)

**`Icons/` — Application Icons:**
- Purpose: App icon resources for packaging and display
- Contains: `stock.ico` — used for window, taskbar, installer, and shortcut icons

**Root files — Configuration & Metadata:**
- `.env` — Environment variables (DB credentials, API config, SSL paths)
- `package.json` — Electron app definition, build config, npm scripts
- `REVIEW.md` — Code review documentation

## Key File Locations

**Entry Points:**
- `src/main.js`: Electron desktop app entry — browser window creation, IPC handler registration, auto-updater setup
- `server/index.js`: Express API server entry — SSL bootstrap, HTTP/HTTPS listener, master DB init
- `src/index.html`: Renderer HTML shell — loads `renderer.js`, defines app layout and PIN gate UI
- `src/setup.html`: Standalone setup page for configuring server URL and company name

**Configuration:**
- `package.json`: Electron + electron-builder config (appId, nsis, publish, icon, asarUnpack)
- `server/package.json`: Server-only dependencies (express, pg, bcrypt, speakeasy, qrcode)
- `src/config.js`: Simple module exporting `API_URL` constant used by main process
- `.env`: Runtime environment variables (DB_HOST, DB_PORT, DB_USER, DB_PASSWORD, API_URL, SSL_*, DB_SSL, ALLOWED_ORIGINS)

**Core Logic:**
- `src/renderer.js`: Full UI logic — PIN gate flow, tab system, form CRUD, CSV/PDF export, status polling, auto-updater UI, local state management (909 lines)
- `src/main.js`: Electron main process — 20+ IPC handlers, JSON file store factories, PDF export with hidden BrowserWindow, auto-updater events (341 lines)
- `src/server.js`: Express REST API — 10 route handlers, multi-tenant PIN authentication, TOTP 2FA, entry CRUD with validation, rate limiting (558 lines)
- `src/db/pool.js`: PostgreSQL connection pooling — master pool (max 3), tenant pools (max 2 each), schema auto-creation (120 lines)
- `src/preload.js`: Context bridge — exposes 25+ methods from main process to renderer via `window.stockApi` (26 lines)

**Testing:**
- No test files detected. `package.json` has `"test": "node --test"` script and `"supertest": "^7.2.2"` dev dependency but no test files found.

**Styling:**
- `src/styles.css`: Complete application stylesheet — dark gradient background, PIN gate layout, metric cards, tables, entry form, status indicators, update banner, responsive breakpoints (624 lines)

## Naming Conventions

**Files:**
- `kebab-case` for most files: `stock-data.json`, `stock-pin.json`, `app-config.json`, `stock-entries`, `stock-balance`
- `snake_case` for DB tables and columns: `stock_entries`, `tenant_id`, `pin_hash`, `company_name`, `totp_secret`, `created_at`
- `camelCase` for JS identifiers and IPC channels: `createWindow`, `handlePinLogin`, `pin:save-local`, `data:load-local`
- `PascalCase` for classes/types: Not used (no TypeScript or classes in this project)

**Directories:**
- All lowercase short names: `src/`, `db/`, `build/`, `scripts/`, `dist/`, `server/`

**IPC Channel Names:**
- Namespaced with colons: `config:get-api-url`, `config:get`, `config:save`, `pin:save-local`, `pin:load-local`, `pin:verify`, `pin:status`, `data:load-local`, `data:save-local`, `stock:export-report-pdf`, `update:check`, `update:download`, `update:install`, `app:version`
- Domain prefix: `config:*`, `pin:*`, `data:*`, `stock:*`, `update:*`, `app:*`

**API Routes:**
- RESTful with `/api/` prefix: `/api/health`, `/api/pin`, `/api/pin/verify`, `/api/pin/status`, `/api/auth/totp-verify`, `/api/entries`, `/api/stock`
- `snake_case` JSON keys: `tenant_id`, `company_name`, `device_name`, `device_token`, `totp_enabled`, `totpQrCode`, `x-access-pin`, `x-device-token`
- `camelCase` in stock entry objects: `createdAt` (mapped from `created_at` in SQL query aliases)

## Where to Add New Code

**New Feature (e.g., new report type, new tab):**
- Primary UI logic: `src/renderer.js` — add new tab handler, render function, event binding
- HTML structure: `src/index.html` — add new tab button and view section
- IPC channel: `src/main.js` — add `ipcMain.handle()` for new channel
- Bridge method: `src/preload.js` — add method to `stockApi` exposure
- API endpoint: `src/server.js` — add Express route (if server-side logic needed)
- Database: `src/db/schema.sql` — add new table or migration steps

**New Component/Module (e.g., new domain entity):**
- Server-side logic: `src/server.js` — add route handlers near existing ones (~line 431 pattern)
- Or extract to a new file in `src/` if complex (e.g., `src/invoices.js`)
- Database pool: `src/db/pool.js` uses shared pools — add new queries via existing pool exports

**New Server Middleware:**
- Add to `src/server.js` — register with `app.use()` before route definitions (line 22-31 pattern)
- For auth middlewares, follow `resolveTenant` pattern at line 148

**Utilities:**
- Simple config exports: `src/config.js` (line 2 pattern)
- Shared helpers used in main process: add to `src/main.js` or a new file in `src/`
- Shared helpers used in server: add to appropriate module in `src/` or `src/db/`

**Database Changes:**
- Table schema changes: update `TENANT_SCHEMA` constant in `src/server.js` (line 88-113)
- Master table changes: update `ensureMasterDb()` SQL in `src/db/pool.js` (line 76-118)
- Reference schema: update `src/db/schema.sql` to match
- Migration scripts: add new file in `src/db/` or `scripts/` following `migrate.js` pattern
