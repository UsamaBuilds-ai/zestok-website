    Create the mobile app source with three files:

    **`mobile/index.html`** — Entry point HTML page:
    - DOCTYPE html5, lang="en"
    - Viewport meta with width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no (prevents zoom on form input)
    - apple-mobile-web-app-capable meta for full-screen on iOS (future-proofing)

    - Title: "Zestok"
    - Link to style.css (relative path)
    - Script src/main.js (type="module", defer)
    - Body structure:
      - `<div id="app">` container
        - `<header>` with app title "Zestok"
        - `<main>` section with connection status display:
          - Status indicator circle (green/red/gray)
          - Status text label
          - Server URL display
          - A "Retry" button
    - Safe-area padding via env(safe-area-inset-*) for notched devices

    **`mobile/src/main.js`** — Application entry point:
    - const API_BASE = 'http://84.235.249.239:3000'
    - async function checkHealth():
      - Sets indicator to "checking" (gray + spinner animation class)
      - Sets status text to "Checking server..."
      - Uses fetch(\`${API_BASE}/api/health\`) for the health check request
      - On OK response (200): set indicator "connected" (green), status text "Connected"
      - On non-OK response: set indicator "error" (red), status text "Server error: {status}"
      - On network error: set indicator "error" (red), status text "Server unreachable"
      - Includes a 5-second timeout on the fetch request via AbortController
    - document.addEventListener('DOMContentLoaded', checkHealth)
    - Wire the "Retry" button to call checkHealth()
    - On Android, CapacitorHttp patches fetch to use native HTTP (OkHttp) which bypasses CORS

    **`mobile/src/style.css`** — Mobile-first styling:
    - CSS reset (box-sizing border-box, margin 0, padding 0)
    - :root CSS custom properties for theming:
      - --bg-primary: #1a1a2e (dark navy)
      - --bg-secondary: #16213e
      - --text-primary: #e0e0e0
      - --accent-green: #4caf50
      - --accent-red: #f44336
      - --accent-gray: #9e9e9e
    - html, body: height 100%, font-family system-ui sans-serif, bg-primary, text-primary
    - #app: min-height 100dvh, flexbox column layout
    - header: centered text, padding 24px 16px, font-size 1.25rem, font-weight 600
    - main: flex-grow 1, flexbox centered column, gap 24px, padding 16px
    - Status indicator circle: 80px diameter, border-radius 50%, transition background-color 0.3s
      - .checking: gray with subtle spin animation (CSS keyframe rotate)
      - .connected: green with glow box-shadow
      - .error: red
    - Status label: font-size 1.125rem, text-align center
    - Server URL: font-size 0.75rem, color rgba(224,224,224,0.6), text-align center
    - Retry button: 48px min-height (44px+ touch target), border-radius 8px, bg-secondary, text-primary, font-size 1rem, cursor pointer, hover/active states
    - @keyframes spin: 0% to 360% rotation for checking indicator
    - Safe-area padding using env(safe-area-inset-*) on body padding

<task type="auto">
  <name>Task 2: Build web assets, initialize Android platform, configure manifest</name>
  <files>
    mobile/dist/
    mobile/android/
    mobile/android/app/src/main/AndroidManifest.xml
  </files>
  <action>
    From the `mobile/` directory, execute in sequence:

    Step 1 — Build web assets with Vite:
    ```powershell
    cd mobile
    npx vite build
    ```
    This creates `mobile/dist/` with index.html, assets/main.js, assets/style.css

    Step 2 — Add Android platform:
    ```powershell
    npx cap add android
    ```
    This reads capacitor.config.ts and creates `mobile/android/` with a complete Android Studio project.
    If npx cap add android prompts for confirmation, pipe 'y' or use auto-confirm flags.

    Step 3 — Verify Android manifest:
    After cap add android, verify that `mobile/android/app/src/main/AndroidManifest.xml` contains:
    - `android:usesCleartextTraffic="true"` on the `<application>` element
    - `<uses-permission android:name="android.permission.INTERNET" />`
    
    If either is missing, add it manually. The cleartext flag should be auto-generated from capacitor.config.ts server.cleartext: true (D-05), and INTERNET is added by default. This step is a defensive check.

    Step 4 — Sync Capacitor config to Android:
    ```powershell
    npx cap sync android
    ```
    This copies capacitor.config.ts settings to the Android project and copies web assets from dist/ to android/app/src/main/assets/public/
  </action>
