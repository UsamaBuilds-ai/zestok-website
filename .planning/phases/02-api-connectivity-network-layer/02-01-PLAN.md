<objective>
Configure Android cleartext HTTP specifically for the API server, implement the centralized API service layer, and wire up the auth header skeleton. This plan establishes the secure networking foundation and the request infrastructure that all subsequent phases build on.

Purpose: Android 9+ blocks cleartext HTTP by default. The existing `usesCleartextTraffic="true"` blanket permission is too permissive. This plan replaces it with a targeted `network_security_config.xml` that allows cleartext only to the known API server IP, and implements the shared apiRequest() wrapper that every backend call will use.

</objective>

    Create `mobile/src/api.js` with the following structure, implementing ALL decisions D-14 through D-18 and D-22:

    ```javascript
    const API_BASE = 'http://84.235.249.239:3000/api';
    const DEFAULT_TIMEOUT = 10000;

    export function getAuthHeaders() {
      // Phase 2: skeleton — returns empty object.
      // Phase 3 will populate from Preferences:
      //   { 'x-access-pin': pin, 'x-device-token': deviceToken }
      return {};
    }

    export async function apiRequest(path, options = {}) {
      const { method = 'GET', body, timeout = DEFAULT_TIMEOUT, auth = false, headers: extraHeaders = {} } = options;

      const controller = new AbortController();
      const timeoutId = setTimeout(() => controller.abort(), timeout);

      const headers = {
        'Content-Type': 'application/json',
        ...extraHeaders,
      };

      if (auth) {
        Object.assign(headers, getAuthHeaders());
      }

      try {
        const res = await fetch(`${API_BASE}${path}`, {
          method,
          headers,
          body: body ? JSON.stringify(body) : undefined,
          signal: controller.signal,
        });
        clearTimeout(timeoutId);

        let data;
        try {
          data = await res.json();
        } catch {
          data = null;
        }

        if (res.ok) {
          return { ok: true, data, status: res.status };
        }

        return { ok: false, data, error: 'server_error', status: res.status };
      } catch (err) {
        clearTimeout(timeoutId);

        if (err.name === 'AbortError') {
          return { ok: false, error: 'network_error', status: 0 };
        }

        return { ok: false, error: 'network_error', status: 0 };
      }
    }

    export async function getHealth() {
      return apiRequest('/health');
    }

    export async function getEntries(options = {}) {
      return apiRequest('/entries', { ...options, auth: true });
    }

    export async function verifyPin(pin) {
      return apiRequest('/pin/verify', {
        method: 'POST',
        body: {},
        headers: { 'x-access-pin': pin },
      });
    }
    ```

    Implementation notes:
    - D-14: Single apiRequest() wrapper — all logic centralized
    - D-15: API_BASE = 'http://84.235.249.239:3000/api' (includes /api prefix, matching server route structure)
    - D-16: AbortController with 10s default timeout, overridable per-call via options.timeout
    - D-17: Structured response — { ok, data, error, status } for all paths; network errors return { ok: false, error: 'network_error', status: 0 }; HTTP errors return { ok: false, error: 'server_error', status }
    - D-18: Per-endpoint convenience functions: getHealth(), getEntries(), verifyPin(pin)
    - D-22: apiRequest accepts options.auth flag; when truthy, injects headers from getAuthHeaders(). Phase 2 returns {} — Phase 3 populates via Preferences

    IMPORTANT: Use plain fetch() (not CapacitorHttp import). CapacitorHttp plugin is already enabled in capacitor.config.ts with CapacitorHttp.enabled: true — this patches the global fetch() to use native OkHttp on Android. No import changes needed.
