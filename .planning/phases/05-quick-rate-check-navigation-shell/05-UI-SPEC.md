## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tab gap (nav-icon to nav-label gap), inner card padding |
| sm | 8px | Dropdown item padding, nav tab padding top/bottom |
| md | 16px | Default element spacing, settings item padding horizontal, content padding |
| lg | 24px | Settings section margin-top (sign-out button), card group gaps |
| xl | 32px | Sign-out button margin-top |
| 2xl | 48px | Settings item min-height, sign-out button height |
| 3xl | 64px | Bottom nav height (content area padding-bottom to clear nav) |

**Exceptions:**
- Bottom nav total height: ~64px (padding: 8px 0 + nav-icon 1.25rem + nav-label 0.625rem + gap 2px). Safe-area padding-bottom via `env(safe-area-inset-bottom)`.
- Content area (`#dashboard-view`, `#ratecheck-view`, `#settings-view`) uses `padding-bottom: calc(16px + env(safe-area-inset-bottom))` to avoid being covered by bottom nav (RESEARCH.md Pitfall 3: z-index 90, retry bar z-index 100).
- Settings item `padding: 16px` — matches `.metric-card` padding from Phase 4.
- Settings sign-out button `height: 48px` — matches 44px+ touch target requirement from UI-02.
- Touch targets: all nav tabs, sign-out button, dropdown items, health check retry button maintain 44px+ min-height per D-02/UI-02.

**Source:** CONTEXT.md D-53/D-54 (nav bar), RESEARCH.md §Code Examples (nav CSS at 64px, button at 48px). [Pre-populated from upstream]

## Interaction Contracts

### I-01: Tab Switching

| Step | Action |
|------|--------|
| 1 | User taps bottom nav tab button (`.nav-tab[data-tab="dashboard"|"ratecheck"|"settings"]`) |
| 2 | `switchTab(tabName)` is called |
| 3 | All 3 views have `.hidden` class added: `#dashboard-view`, `#ratecheck-view`, `#settings-view` |
| 4 | Target view has `.hidden` class removed |
| 5 | Header tab-name text updated via `#header-tab-name.textContent` (preserving `#network-badge` DOM node) |
| 6 | Active nav tab updated: `document.querySelectorAll('.nav-tab').forEach(btn => btn.classList.toggle('active', btn.dataset.tab === tabName))` |
| 7 | View show function called: `showDashboard()`, `showRateCheck(getBalancesState())`, or `showSettings()` |

**Edge cases:**
- If dashboard data not yet loaded when switching → dashboard's existing `loadDashboard()` handles the load
- If rate check has no `_balances` data → shows empty state prompt (no error)
- If Settings health check fails → shows "Unreachable" status (not an error overlay)
- Debounce timer must be cleared in hide functions (Pitfall 6 from RESEARCH.md)

### I-02: Rate Check Autocomplete

| Step | Action |
|------|--------|
| 1 | User types in `#ratecheck-input` |
| 2 | `input` event fires → clear any existing `_debounceTimer` |
| 3 | If input is empty → hide dropdown, hide result, clear selected item, return immediately (instant clear per D-48 pattern) |
| 4 | Set `_debounceTimer = setTimeout(fn, 300)` |
| 5 | After 300ms: filter `_balances` using `keyFor(item.item).includes(keyFor(input.value))` |
| 6 | Render dropdown: if matches found → show items with name (bold) + category (small, muted); if no matches → show "No items found" |
| 7 | User taps dropdown item → `click` event → `selectItem()`: fill input value, hide dropdown, show result cards with rate + balance |
| 8 | User taps outside → `document touchstart` listener hides dropdown |

**Keyboard:** Input does NOT have `autocomplete="on"` — uses custom dropdown. Escape key closes dropdown (not required but recommended).

### I-03: Settings Health Check

| Step | Action |
|------|--------|
| 1 | `showSettings()` called → set health status to "Checking..." show spinner |
| 2 | Check `isConnected()` if available |
| 3 | If offline → show "Offline", no API call |
| 4 | If online → call `getHealth()` |
| 5 | On success (result.ok) → show "Connected" |
| 6 | On failure → show "Unreachable" |
| 7 | Retry button (if implemented) → reset to "Checking..." and re-call step 2 |

### I-04: Sign-Out Flow

| Step | Action |
|------|--------|
| 1 | User taps "Sign Out" button in Settings |
| 2 | `signOut()` in auth.js runs (no confirmation dialog, per D-57) |
| 3 | `await Preferences.remove({ key: 'accessPin' })` — forces PIN re-entry on next start |
| 4 | `await Preferences.remove({ key: 'biometricEnabled' })` — prevents biometric on resume (per D-58) |
| 5 | `pinHash` preserved — enables offline PIN re-entry (per D-57) |
| 6 | `cachedEntries` preserved — keeps offline data (per D-57) |
| 7 | `_notify({ isAuthenticated: false, companyName: '', tenantId: '' })` |
| 8 | `onAuthChange` subscriber in main.js detects `isAuthenticated === false` → calls `showPinGate()` |
| 9 | PIN gate overlay appears, app returns to unauthenticated state |
