<action>
Create `mobile/src/balances.js` as a pure ES module with NO side effects, no DOM, no imports from other project modules.

Export two functions:

1. `keyFor(value)` — trims, lowercases, converts to string. Used for case-insensitive comparison matching. Pattern from renderer.js:441.

2. `getBalances(entries)` — exact port from renderer.js:424-453:
   - Create a `Map` keyed by `keyFor(entry.item)`
   - For each entry, accumulate: `inQty += entry.quantity` if type==="in", `outQty += entry.quantity` if type==="out"
   - On "in" entries, set `latestRate = entry.rate` (the most recent "in" rate becomes the current rate for value calculation — matches the SQL stock_balance view logic)
   - Set `category = entry.category || current.category || "-"`
   - Compute `balance = inQty - outQty`
   - Compute `value = balance * latestRate`
   - Return `Array.from(items.values()).sort((a, b) => a.item.localeCompare(b.item))` — sorted alphabetically by item name

Each balance item object shape: `{ item, category, inQty, outQty, balance, latestRate, value }`

Note: This is a PURE function. Do NOT add any state or side effects. The caller supplies the entries array.
</action>
