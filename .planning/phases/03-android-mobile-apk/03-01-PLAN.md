    Create the two service modules that every component will import. The API client wraps fetch with timeout, defines type interfaces for stock and PIN responses, and exports typed functions. The storage service uses Capacitor Preferences for persistent key/value storage.

    **Step 1 — Create mobile/src/services/api.ts:**
    
    Define TypeScript interfaces matching the expected API response shapes (RESEARCH.md Code Examples):
    
    ```typescript
    const TIMEOUT_MS = 5000;

    export interface StockItem {
      item: string;
      category: string;
      inQty: number;
      outQty: number;
      balance: number;
      latestRate: number;
    }

    export interface StockResponse {
      [category: string]: StockItem[];
    }

    export interface PinStatusResponse {
      configured: boolean;
      message?: string;
    }

    export interface PinVerifyResponse {
      valid?: boolean;
      message?: string;
    }
    ```

    Define the custom ApiError class with type, statusCode, and message:

    ```typescript
    export class ApiError extends Error {
      constructor(
        public type: 'network' | 'timeout' | 'http',
        public statusCode?: number,
        message?: string
      ) {
        super(message || type);
        this.name = 'ApiError';
      }
    }
    ```

    Implement fetchWithTimeout — an AbortController-based wrapper (RESEARCH.md Pattern 7):
    - TIMEOUT_MS = 5000 (5 seconds — fail fast for mobile)
    - AbortController with setTimeout for timeout detection
    - Catch AbortError → throw ApiError('timeout')
    - Catch general network errors → throw ApiError('network')
    - Final block: clearTimeout

    Export four async functions:

    **fetchStock(serverIP: string, pin: string): Promise<StockResponse>**
    - GET `http://${serverIP}:3000/api/stock`
    - Header: `x-access-pin: ${pin}`
    - Not OK: throw ApiError('http', status)
    - OK: return response.json()

    **verifyPin(serverIP: string, pin: string): Promise<PinVerifyResponse>**
    - GET `http://${serverIP}:3000/api/pin/verify`
    - Header: `x-access-pin: ${pin}`
    - Not OK (401): throw ApiError('http', 401, 'Incorrect PIN')
    - OK: return response.json()

    **getPinStatus(serverIP: string): Promise<PinStatusResponse>**
    - GET `http://${serverIP}:3000/api/pin/status`
    - No auth header (status check is unauthenticated)
    - Not OK: throw ApiError('http', status)
    - OK: return response.json()

    **testConnection(serverIP: string): Promise<boolean>**
    - GET `http://${serverIP}:3000/api/pin/status`
    - Returns true if response.ok, false on any error (network, timeout, HTTP)
    - This is a lightweight ping — catches all errors gracefully

    **Step 2 — Create mobile/src/services/storage.ts (RESEARCH.md Pattern 3):**

    ```typescript
    import { Preferences } from '@capacitor/preferences';

    const STORAGE_KEYS = {
      SERVER_IP: 'stock_server_ip',
    };

    export async function getServerIP(): Promise<string | null> {
      const { value } = await Preferences.get({ key: STORAGE_KEYS.SERVER_IP });
      return value;
    }

    export async function setServerIP(ip: string): Promise<void> {
      await Preferences.set({ key: STORAGE_KEYS.SERVER_IP, value: ip });
    }

    export async function clearServerIP(): Promise<void> {
      await Preferences.remove({ key: STORAGE_KEYS.SERVER_IP });
    }
    ```

    **IMPORTANT AVOIDANCES:**
    - Do NOT use `localStorage` for server IP — use Capacitor Preferences (per D-03, Research Don't Hand-Roll: "OS may clear localStorage on low storage")
    - Do NOT use `type: 'network' | 'http'` string literals without the ApiError class — the type discriminator is essential for components to handle errors differently (network → Retry, http 401 → wrong PIN, http other → unexpected error)
    - Do NOT import React-dependent code in api.ts — it's a pure TypeScript service that can be tested without DOM
    - Do NOT store the PIN in api.ts state — PIN is passed as a parameter each call (per D-07: PIN never stored on device)
    - Do NOT hardcode the server IP in api.ts — it is passed as a parameter from the component that reads it from storage
    - Do NOT use `fetch(url, { signal: controller.signal })` without the `catch (error.name === 'AbortError')` guard — not all browsers throw the same AbortError shape
  </action>

    **Step 3 — Create mobile/src/App.tsx (Root component with state machine, lifecycle, routing):**

    App.tsx manages the entire app state as a finite state machine with three states:
    1. `'loading'` — Initial state, checking storage for saved IP
    2. `'setup'` — No saved IP, show ServerSetup
    3. `'pin'` — Saved IP exists but not authenticated, show PinScreen
    4. `'stock'` — Authenticated, show StockList (the StockList component is imported and rendered when this state is active)
    5. `'error'` — IP saved but server unreachable (D-21), show error with Retry or Edit IP

    State variables:
    - `appState`: 'loading' | 'setup' | 'pin' | 'stock' | 'error'
    - `serverIP`: string | null (from storage)
    - `isAuthenticated`: boolean (in-memory, reset on app kill — D-07, D-10)
    - `errorMessage`: string | null

    Lifecycle handler (RESEARCH.md Pattern 1):
    ```tsx
    useEffect(() => {
      // Check for saved server IP on mount
      const initApp = async () => {
        const ip = await getServerIP();
        if (ip) {
          setServerIP(ip);
          // Test connection (D-05 style)
          const connected = await testConnection(ip);
          if (connected) {
            setAppState('pin');
          } else {
            setAppState('error');
            setErrorMessage('Cannot connect to server');
          }
        } else {
          setAppState('setup');
        }
      };
      initApp();
    }, []);
    ```

    App.resume listener (D-08, D-16):
    ```tsx
    useEffect(() => {
      const handler = App.addListener('appStateChange', ({ isActive }) => {
        if (isActive) {
          // App came to foreground — require PIN re-entry (D-08)
          if (appState === 'stock') {
            setIsAuthenticated(false);
            setAppState('pin');
          }
        }
      });
      return () => { handler.then(h => h.remove()); };
    }, [appState]);
    ```

    Render function dispatches based on appState:
    ```tsx
    const renderScreen = () => {
      switch (appState) {
        case 'loading':
          return <div className="loading-overlay"><IonSpinner name="crescent" /></div>;
        case 'setup':
          return <ServerSetup
            onComplete={(ip) => { setServerIP(ip); setAppState('pin'); }}
          />;
        case 'pin':
          return <PinScreen
            serverIP={serverIP!}
            onSuccess={() => { setIsAuthenticated(true); setAppState('stock'); }}
            onEditIP={() => { clearServerIP(); setAppState('setup'); }}
          />;
        case 'stock':
          return <StockList />;  // Will be fully built in Plan 03-02
        case 'error':
          return <ErrorState
            message={errorMessage || 'Cannot connect to server'}
            onRetry={async () => {
              const connected = await testConnection(serverIP!);
              if (connected) setAppState('pin');
              else setErrorMessage('Cannot connect to server');
            }}
            onEditIP={() => { clearServerIP(); setAppState('setup'); }}
          />;
      }
    };
    ```

    The App component wraps everything in `<IonApp>`.

    ```tsx
    useEffect(() => {
      if (!isLocked) return;
      const interval = setInterval(() => {
        setLockoutRemaining(prev => {

            clearInterval(interval);
            setIsLocked(false);
            setAttemptCount(0);
            return 30;
          }
          return prev - 1;
        });

      return () => clearInterval(interval);
    ```
