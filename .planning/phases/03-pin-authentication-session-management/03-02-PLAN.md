<objective>
Implement session persistence and app lifecycle resume handling — the second vertical slice of PIN authentication. After successful online PIN verification, the raw PIN, company name, and tenant ID are persisted in `@capacitor/preferences`. The user must re-enter their PIN on every fresh app start (no auto-restore, per D-27). When the app resumes from background, the `appStateChange` listener re-shows the PIN gate for re-authentication (no timeout threshold, per D-30). The stored PIN in Preferences feeds into `getAuthHeaders()` so all subsequent authenticated API calls can inject the `x-access-pin` header.

Purpose: Session persistence is critical because the server requires the raw PIN via `x-access-pin` header for every authenticated API call — entries, stock data, etc. Without this, no Phase 4+ feature can make authenticated requests. The lifecycle handler ensures the app re-authenticates on resume, preventing unauthorized access after the phone is locked or put down.

Output: Updated mobile/src/auth.js (session persistence, lifecycle listener), updated mobile/src/main.js (lifecycle registration), updated mobile/src/api.js (async getAuthHeaders from Preferences)
</objective>
