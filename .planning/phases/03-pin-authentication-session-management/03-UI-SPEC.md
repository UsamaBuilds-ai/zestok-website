### Detailed Interaction Rules

**1. PIN Gate Visibility**
- **On app start:** PIN gate is shown immediately as a full-screen overlay — user sees nothing else until authenticated
- **On app resume (from background):** PIN gate is re-shown as a full-screen overlay over whatever view was last active (D-29, D-30)
- **On successful verification:** PIN gate slides up / fades out (0.3s ease) to reveal the dashboard behind it
- **On sign-out (future Phase 5):** PIN gate re-appears

**2. Input Behavior**
- **Auto-focus:** Input field receives focus immediately when PIN gate appears — numeric keypad opens automatically on mobile (D-24)
- **Input mode:** `inputmode="numeric"` + `pattern="[0-9]*"` → hardware numeric keypad with no decimal/symbol keys (D-23)
- **Masking:** `type="password"` or CSS `-webkit-text-security: disc` — all digits shown as dots/asterisks (D-24)
- **Length validation:** Accepts 4-6 digits. If user presses Enter/Go with <4 digits, do NOT submit — show "PIN must be 4-6 digits" inline error. If >6 digits, silently truncate to 6.
- **Max length:** HTML `maxlength="6"` attribute prevents more than 6 characters from being entered

**3. Submission**
- **Trigger:** Keyboard Enter/Go key press activates verification (D-25)
- **No submit button:** Do NOT render a submit/confirm button for this phase (D-25)
- **Debounce:** Prevent re-submission while a verification request is in-flight (D-34 spinner prevents interaction)

**4. Loading State (D-34)**
- **Overlay type:** Full-screen fixed-position overlay with semi-transparent dark `rgba(0,0,0,0.7)` backdrop
- **Content:** Centered 48px spinning indicator (reuse existing `@keyframes spin` animation from `style.css` with a `border-top-color: var(--text-primary)` circular spinner) + "Verifying..." text below
- **Duration:** Visible from submission until server responds or timeout fires (10s timeout via AbortController)
- **Dismissal:** On resolve — overlay fades out. On timeout/reject — overlay hides, error state shown
- **Interaction blocked:** All clicks pass-through blocked by the overlay — user cannot interact with anything behind it

**5. Error States**

| Error | Visual Treatment | Duration | Recovery |
|-------|-----------------|----------|----------|
| Invalid PIN (D-35) | Inline red text below input (`color: var(--accent-red)`). Input border changes to `rgba(244,67,54,0.5)`. Input value cleared. | Persistent until next successful submission or input change | User re-enters PIN and presses Enter |
| Rate limited 429 (D-37) | Same inline red error treatment as invalid PIN. Copy: "Too many attempts. Try again later." | Persistent until rate limit window expires (15 min) | User must wait. No retry button on this error. |
| Network unreachable (D-36) | Retry bar at screen bottom (reuses Phase 2 `#retry-bar` pattern) — red bar with "Cannot reach server" + "Retry" button | Persistent until dismissed by retry or server becomes reachable | User taps "Retry" → re-attempts verification |
| Offline fallback — mismatch (D-38) | Same inline red error as invalid PIN. Copy: "Invalid PIN. Try again." (same as online) | Until successful local match | User re-enters PIN |

**6. Offline Fallback Notification (D-32)**
- When `isConnected()` returns false, show a brief non-blocking toast above the input: "Server unreachable. Using stored credentials."
- Toast auto-dismisses after 3 seconds or on first input interaction
- Does NOT block the input — user can type immediately

**7. App Lifecycle — Resume (D-29, D-30)**
- `App.addListener('appStateChange', ...)` registered during app init
- On `isActive === true` (foreground): immediately show PIN gate overlay
- PIN gate replaces whatever view was visible before backgrounding
- No timeout threshold — every resume triggers PIN re-entry

**8. Session Persistence Background (D-26, D-27)**
- On successful online verify: store `accessPin`, `companyName`, `tenantId`, `pinHash` in Preferences
- On fresh app start: NO auto-restore of session — always show PIN gate
- The stored `accessPin` is used by `getAuthHeaders()` for subsequent API calls, NOT for bypassing the PIN gate
