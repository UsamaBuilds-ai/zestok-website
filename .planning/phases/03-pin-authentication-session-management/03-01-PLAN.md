<objective>
Build the PIN gate UI and core online verification flow — the first vertical slice of PIN authentication. When the app launches, the user sees a numeric PIN entry overlay, enters their PIN, and the app verifies it against the server via `GET /api/pin/verify`. On success, the gate hides and the dashboard appears. On failure, an inline error is shown.

Purpose: This is the primary entry point for the entire app. Without this working, nothing else is accessible. The PIN gate is the first thing the user sees on every launch and every resume — getting the UI, verification, and state management right is the foundation for all subsequent auth work.

Output: mobile/src/auth.js (auth state manager), updated mobile/src/api.js (fixed verifyPin, async getAuthHeaders), updated mobile/src/main.js (auth bootstrap), updated mobile/index.html (PIN gate overlay + spinner overlay), updated mobile/src/style.css (PIN gate styles)
</objective>
