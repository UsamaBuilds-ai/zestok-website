<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Express server ↔ main process | Same Node.js process — IPC not used for server control, direct function calls |
| renderer → preload → main | Existing isolated IPC bridge (contextIsolation: true) |
| server.js → filesystem (JSON backup) | Write to userData directory — same trust level as existing JSON file storage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01-01 | Spoofing | server:status IPC | accept | Status information is non-sensitive (server running/stopped) — no authentication needed |
| T-01-01-02 | Tampering | JSON backup file write | mitigate | Write to userData directory (Electron sandboxed path), non-blocking with catch — cannot crash the main request flow |
| T-01-01-03 | Information Disclosure | Server/DB status in footer | accept | Internal network info displayed on desktop UI only — no external exposure |
| T-01-01-04 | Denial of Service | Pool.query in status check | mitigate | SELECT 1 is a no-op query, connectionTimeoutMillis=2000 ensures fast failure; status check is independent of request flow |
</threat_model>
